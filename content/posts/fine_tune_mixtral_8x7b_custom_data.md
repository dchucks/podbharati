### Part 3: Creating Formatted Prompts for Training


**3.1 Define Prompt Creation Function**

To align the dataset inputs with our model's needs, we format the `prompt` and `response` into a single structured input.

```python

def create_prompt(sample):

    bos_token = "<s>"

    system_message = "[INST]Use the provided input to create an instruction that could have been used to generate the response with an LLM."

    response = sample["prompt"].replace("\n\n### Instruction\n", "").replace("\n### Response\n", "").strip()

    input = sample["response"]

    eos_token = "</s>"

    full_prompt = f"{bos_token}{system_message}\n{input}[/INST]{response}{eos_token}"

    return full_prompt

```


**Explanation:**

This function constructs a formatted prompt that combines both `prompt` and `response` from the dataset, embedding them within special tokens (`<s>` and `</s>`) that signal the start and end of a sequence to the model. The `[INST]` tag is used to indicate that the text following it should be treated as an instruction generated by the model.

### Part 4: Model Configuration and
 
Initialization

**4.1 Load the Base Model**

We then proceed to load the Mixtral 8x7B model, configured for optimal performance with our dataset.


```python

from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig

import torch

model_id = "mistralai/Mixtral-8x7B-v0.1"

nf4_config = BitsAndBytesConfig(

    load_in_4bit=True,

    bnb_4bit_quant_type="nf4",

    bnb_4bit_use_double_quant=True,

    bnb_4bit_compute_dtype=torch.bfloat16

)

  

model = AutoModelForCausalLM.from_pretrained(

    model_id,

    device_map='auto',

    quantization_config=nf4_config,

    use_cache=False,

    attn_implementation="flash_attention_2"

)

tokenizer = AutoTokenizer.from_pretrained(model_id)

tokenizer.pad_token = tokenizer.eos_token

tokenizer.padding_side = "right"

```

**Explanation:**

- **AutoModelForCausalLM**: Loads a causal language model from Hugging Face's model hub.

- **BitsAndBytesConfig**: Configures the model to use 4-bit quantization, which helps reduce memory usage while maintaining model performance.

- **Device map**: Automatically assigns model layers to available GPUs for efficient training.

### Part 5: Training the Model

**5.1 Configure Training Parameters**

Setting up the training arguments specifies how the model should be trained, including batch size, learning rate, and the number of training steps.

```python

from transformers import TrainingArguments


args = TrainingArguments(

    output_dir="Mixtral_Alpace_v2",

    max_steps=1000,

    per_device_train_batch_size=32,

    learning_rate=2.5e-5,

    bf16=True

)

```

**5.2 Initialize the Trainer**

With the model and training configuration ready, we initialize the trainer that will manage the fine-tuning process.

```python

from trl import SFTTrainer

max_seq_length = 1024
```

**Explanation:**

- **SFTTrainer**: A specialized trainer for fine-tuning models using self-supervised techniques.

- **Max_seq_length**: Limits the length of input sequences to ensure efficient processing.

Now, we're set to start the training. This step will adjust the model weights based on the custom data provided.

### Part 6: Generating Responses and Evaluating the Model

**6.1 Test the Fine-Tuned Model**

To see how well our model performs, we generate responses using the fine-tuned model.


```python

def generate_response(prompt, model):

    encoded_input = tokenizer(prompt, return_tensors="pt", add_special_tokens=True)

    model_inputs = encoded_input.to('cuda')

  

    generated_ids = model.generate(**model_inputs,

                                   max_new_tokens=150,

                                   do_sample=True,

                                   pad_token_id=tokenizer.eos_token_id)

  

    decoded_output = tokenizer.batch_decode(generated_ids)

    return decoded_output[0]

```
